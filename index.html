The Participant Lifecycle
End to End

From first ad click to long-term retention — a complete view of how participants move through our clinical study ecosystem.

The Journey

9-Stage Participant Lifecycle

Click any stage to explore channels, system requirements, and dashboard metrics.

1
Lead Capture
Ads, newsletter, referral, organic, self-serve studies
Channels / Touchpoints
  • Meta Ads (Facebook & Instagram)
  • Google Ads / Search
  • Newsletter signup forms
  • Referral links from existing participants
  • Organic search / blog content
  • Self-serve study landing pages
System Requirements
  • Auto-import leads from Meta Ads API
  • Landing page form → participant database sync
  • UTM parameter tracking on all inbound links
  • Referral code generation & tracking
Dashboard Metrics
  • Leads by source & campaign
  • Cost per lead by channel
  • Lead → app install conversion rate
  • Weekly/monthly lead volume trend
2
App Install
OneLink tracking, AppsFlyer attribution
Channels / Touchpoints
  • OneLink deep links from ads & emails
  • App Store / Google Play organic search
  • AppsFlyer attribution for install source
System Requirements
  • AppsFlyer SDK integration for attribution
  • OneLink configuration for all campaigns
  • Install event → participant record creation
  • Deferred deep linking for pre-install referrals
Dashboard Metrics
  • Installs by source / campaign
  • Cost per install
  • Install → account creation rate
  • Attribution accuracy / unattributed installs
3
Browse / Explore
In-app browsing, push notifications
Channels / Touchpoints
  • In-app study catalog
  • Push notifications for new studies
  • Email digests of available studies
  • In-app banners and featured studies
System Requirements
  • Study catalog with filtering & eligibility hints
  • Push notification service (segmented sends)
  • Browse activity tracking (views, clicks, time)
  • Personalized study recommendations
Dashboard Metrics
  • Studies viewed per session
  • Browse → eligibility survey start rate
  • Push notification open rate
  • Most viewed / most clicked studies
4
Eligibility Survey
In-app survey, drop-off monitoring by question
Channels / Touchpoints
  • In-app eligibility questionnaire
  • Email reminder for incomplete surveys
  • Push notification nudge to finish survey
System Requirements
  • Question-level drop-off tracking
  • Disqualification reason logging
  • Survey completion event triggers
  • Partial survey save & resume capability
Dashboard Metrics
  • Survey start → complete rate
  • Drop-off rate by question
  • Disqualification reasons breakdown
  • Average time to complete survey
5
Onboarding
Join study, labs, wearable setup
Channels / Touchpoints
  • In-app onboarding flow
  • Email with lab kit / wearable instructions
  • SMS reminders for manual steps
  • Support chat for setup issues
System Requirements
  • Onboarding checklist with step tracking
  • Lab kit shipment tracking integration
  • Wearable device pairing verification
  • Onboarding timeout alerts (stalled participants)
Dashboard Metrics
  • Onboarding completion rate
  • Average time to fully onboard
  • Drop-off by onboarding step
  • Wearable pairing success rate
6
Active in Study
Compliance monitoring, at-risk flagging
Channels / Touchpoints
  • In-app daily/weekly check-ins
  • Push notifications for task reminders
  • Email progress reports
  • Support escalation for at-risk participants
System Requirements
  • Real-time compliance scoring engine
  • At-risk participant flagging & alerts
  • Automated reminder sequences
  • Wearable data ingestion & validation
Dashboard Metrics
  • Compliance rate (overall & by task)
  • At-risk participant count
  • Average daily engagement score
  • Data completeness by participant
7
Study Completion
Exit survey, rewards, next-study recommendation
Channels / Touchpoints
  • In-app completion celebration screen
  • Exit survey / feedback form
  • Reward notification (payment, gift card)
  • Next-study recommendation prompt
System Requirements
  • Study completion event triggers
  • Automated reward issuance
  • Next-study recommendation engine
  • Exit survey data collection & analysis
Dashboard Metrics
  • Study completion rate
  • Exit survey response rate & scores
  • Reward redemption rate
  • Next-study click-through rate
8
Washout Period
Waiting period between studies, public study nudges
Channels / Touchpoints
  • In-app washout countdown / status
  • Email updates on upcoming studies
  • Public study nudges (non-clinical content)
  • Community engagement content
System Requirements
  • Automated washout period tracking per participant
  • Washout end-date calculation & alerts
  • Study eligibility re-check on washout expiry
  • Public study catalog (non-restricted content)
Dashboard Metrics
  • Participants currently in washout
  • Average washout duration
  • Washout → re-engagement rate
  • Content engagement during washout
9
Re-engagement
Segmented outreach, churn detection, incentives
Channels / Touchpoints
  • Segmented email campaigns
  • Push notification re-engagement sequences
  • SMS outreach for high-value participants
  • Special incentives / bonus offers
System Requirements
  • Churn detection model (inactivity thresholds)
  • Segmented campaign builder
  • Incentive tracking & budgeting
  • A/B testing for re-engagement messages
Dashboard Metrics
  • Churned participant count & trend
  • Re-engagement campaign response rate
  • Reactivation rate (returned to browse)
  • Cost per reactivated participant
↻

Participants loop back to Stage 3: Browse / Explore

The lifecycle is continuous — participants cycle through multiple studies over time.

Sufficient data for study baseline requirements

Roadmap

Implementation Phases

A two-phase rollout from minimum viable to full automation.

Phase 1
Minimum Viable
Now → 4 weeks
  • Data cleanup of test/dummy accounts
  • Naming convention for test accounts
  • Get data into their system
  • Basic segmentation queries (manual pull)
  • Study-level dumps for active recruitment
  • Eligibility drop-off visibility by question
  • Dashboard fixes (spend, source attribution)
Phase 2
Full System
4 weeks → 3 months
  • Automated segmentation dashboard with filters
  • One-click email campaigns from dashboard
  • In-app notifications and drip campaigns
  • Washout period auto-tracking
  • Full participant status tracking
  • Churn detection and re-engagement flows
  • API integration to email platform
